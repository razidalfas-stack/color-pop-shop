# TOKO_BAJUU — KOLOR fashion store

## Stack

- **Framework**: TanStack Start v1 (React 19) — NOT Next.js. No `pages/`, `app/`, or `app/layout.tsx`.
- **Runtime**: Bun (dev) / npm (CI/deploy). Both lockfiles (`bun.lock` + `package-lock.json`) are committed.
- **Routing**: File-based via TanStack Router (`src/routes/`). See `src/routes/README.md` for conventions.
- **CSS**: Tailwind v4 + `tw-animate-css`. Custom theme in `src/styles.css` (KOLOR brand palette, `shadow-pop`, `text-gradient-sunset`).
- **UI**: shadcn/ui (New York style, lucide icons). Components at `src/components/ui/`. Utility `cn()` at `src/lib/utils.ts`.
- **State/Data**: TanStack React Query via `src/router.tsx`. QueryClient is injected into route context.

## Config quirks

- `vite.config.ts` delegates to `@lovable.dev/vite-tanstack-config` — do NOT manually add `tanstackStart`, `viteReact`, `tailwindcss`, `tsConfigPaths` or other plugins it already bundles.
- To deploy to **Vercel**, the `nitro` config block must set `preset: "vercel"` in `vite.config.ts`. A `vercel.json` with `outputDirectory: ".vercel/output"` and `framework: null` is also required — see those files for the exact values.
- `@/` path alias → `./src/`.
- ESLint blocks `import "server-only"` (Next.js convention). Use `*.server.ts` or `@tanstack/react-start/server-only` instead.
- `routeTree.gen.ts` is auto-generated by TanStack Router. Never edit by hand.
- Bun config (`bunfig.toml`) enforces a 24h minimum release age. Bypass list at the bottom of that file.

## Commands

```sh
npm run dev          # dev server (Vite)
npm run build        # production build (generates .vercel/output/ for Vercel)
npm run build:dev    # dev-mode build
npm run preview      # preview production build
npm run lint         # ESLint
npm run format       # Prettier (100 width, trailing commas, double quotes)
```

## Project structure

```
src/
  start.ts         # TanStack Start instance + request middleware
  server.ts        # SSR entry point — custom error wrapping for h3
  router.tsx       # React Router + QueryClient factory
  routes/          # file-based routes (see routes/README.md)
  routeTree.gen.ts # auto-generated — do not edit
  lib/
    utils.ts       # cn() helper (clsx + tailwind-merge)
    error-capture.ts  # captures uncaught errors for SSR recovery
    error-page.ts     # renders fallback 500 HTML page
    lovable-error-reporting.ts  # Lovable error reporter
```

## SSR error handling

Server errors are caught at three layers (server entry → request middleware → route error component) and rendered as a user-friendly HTML page. `src/lib/error-capture.ts` works around h3 swallowing thrown errors into generic JSON 500 responses.
